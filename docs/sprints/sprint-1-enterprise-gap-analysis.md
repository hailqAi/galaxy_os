## Decision before implementation

The current architecture should be extended, not replaced. Keep `Department` as the organizational-unit table, `UserRole` as the role-assignment table, and the shared `UserManagementPolicy` as the target-policy boundary. Add only the missing metadata and modules. Do not add a separate authorization service, arbitrary permission strings, dynamic database columns, impersonation, or Sprint 2 domains.
