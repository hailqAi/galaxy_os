# Document AI boundary

`services/document-ai` is reserved for Sprint 14 PDF extraction, OCR, catalog parsing, tables, and assisted processing. In Sprint 0 it has only health/readiness endpoints and no ERP integration, persistence, queue, AI SDK, or document schema.

Future integration requires authenticated API-to-service calls, explicit file limits, malware and content handling, tenant/project authorization, traceable processing, and a reviewed data-retention policy.
