# Security and permissions

All authorization belongs in the API. Web controls improve usability but never grant or deny authority. Future permissions must protect supplier cost, margin, finance, approvals, and exports; significant actions must produce attributable audit records.

Validate every external input, expose safe errors, keep secrets out of Git, use least-privilege production credentials, and never let a future mobile or integration client bypass the API. Roles, permissions, and audit logs begin in Sprint 1.
